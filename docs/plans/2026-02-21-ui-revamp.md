# UI Revamp Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace all existing React Native screens with the revamped designs from Stitch project `7580322135798196968`, preserving all existing business logic and navigation.

**Architecture:** Each screen's HTML is fetched from Stitch MCP, then translated to React Native + React Native Paper components. A shared theme file captures all design tokens upfront so every screen is consistent. Logic (Supabase calls, Zustand stores, navigation) is kept intact — only the JSX/StyleSheet layer changes.

**Tech Stack:** React Native (Expo), TypeScript, React Native Paper, Zustand, Expo Router, Supabase, Stitch MCP (`mcp__stitch__get_screen`)

**Stitch Project ID:** `7580322135798196968`

**Stitch screen to reference:** `projects/7580322135798196968/screens/db644f60aab34c87878fd30177674006` (Dashboard — most tokens visible here)

**Files:**
- Create: `mobile/lib/theme.ts`
- Modify: `mobile/app/_layout.tsx`

Use `mcp__stitch__get_screen` with name `projects/7580322135798196968/screens/db644f60aab34c87878fd30177674006`, then extract:
- Primary color: `#136dec`
- Background, surface, text, border colors
- Font: Inter (use `expo-font` or system fallback)
- Border radius: 8px base
- Spacing scale

**Step 2: Create `mobile/lib/theme.ts`**

```typescript
import { MD3LightTheme } from 'react-native-paper';

export const theme = {
  ...MD3LightTheme,
  colors: {
    ...MD3LightTheme.colors,
    primary: '#136dec',
    // fill remaining from Stitch extraction
  },
};

export const radius = { sm: 4, md: 8, lg: 16, xl: 24 };
```

**Step 3: Update `mobile/app/_layout.tsx` to use theme**

Wrap `<Stack>` with `<PaperProvider theme={theme}>`.

**Step 4: Verify app still boots**

```bash

```
Expected: no crashes, theme applied globally.

**Step 5: Commit**

